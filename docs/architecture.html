CI4MS Architecture

System Blueprint

Understand the moving parts behind CI4MS—filters, routes, modules, caches, and tooling. Use this guide alongside the developer handbook when designing new features or debugging runtime behaviour.

Application Bootstrap & Request Lifecycle

  • Filters: app/Filters/Ci4ms.php enforces install checks, maintenance redirects, and menu cache hydration in after().
  • Dynamic filter discovery: app/Config/Filters.php scans module and theme filter directories, then merges CSRF exceptions from Modules\Backend\Config\BackendConfig.
  • Routing: app/Config/Routes.php preloads settings, attaches template routes, then imports every module’s Config/Routes.php before declaring front-end routes.

CommonModel Abstraction

bertugfahriozer/ci4commonmodel powers the generic CRUD layer. Modules lean on helpers such as lists, selectOne, create, createMany, edit, remove, and isHave.

The backend BaseController instantiates CommonModel once, exposing it via $this->commonModel to child controllers.

Authentication & Authorization

  • AuthLibrary: Manages login/logout, remember-me cookies, lockouts, password resets, and notification emails. Stores session keys (logged_in, redirect_url) and caches user permissions.
  • Backend filters: Modules\Backend\Filters\BackendAfterLoginFilter enforces authentication, verifies permissions, checks theme assets, and warms cached settings.
  • BaseController duties: Resolves the logged-in user (Modules\Users\Models\UserscrudModel::loggedUser()), navigation (AuthLibrary::sidebarNavigation()), mail config, and $this->defData.
  • Permission tables: auth_permissions_pages (module/page CRUD flags) and auth_users_permissions (user overrides). Modules\Methods provides scanning utilities to sync routes.

Module Pattern

Each module under modules/<Name> typically includes:

  • Config/Routes.php – routes plus metadata (role options).
  • Config/*.php – module-specific settings.
  • Controllers/ – usually extend the backend base controller.
  • Models/, Views/, optional Libraries/, Helpers/, Language/, Filters/, Database/.

Scaffold new modules with php spark make:module Foo (from ci4-cms-erp/ext_module_generator) and follow up with routes, implementation, and permissions.

Installation Flow

Web Installer

  • Module: Modules\Install.
  • Copies env to .env, sets base configuration, runs migrations, and seeds defaults via InstallService.
  • Regenerates app/Config/Routes.php from the command template.

CLI Seeder

  • Ci4msDefaultsSeeder prompts for admin credentials.
  • Calls InstallService::createDefaultData() to seed modules, permissions, admin account, sample content, and settings.

Caching & Configuration

  • Settings: Stored as JSON/string in settings; cached for 24h.
  • Menus: Cached for 24h; invalidated when menu structure updates.
  • User permissions: Cached per user ({userId}_permissions); clear when updating roles.
  • Mail settings: Encrypted array in settings.mail; decrypt through CommonLibrary::phpMailer() or backend base controller.

Theme System

  • Themes live in public/templates/<theme> with optional overrides in app/Config, app/Controllers, etc.
  • Modules\Theme handles ZIP uploads, temporary extraction (writable/tmp), duplicate detection, and final installation.
  • BackendAfterLoginFilter warns administrators if info.xml or screenshot.png is missing.

Content & SEO

  • Home controller: App\Controllers\Home renders front pages and blogs, parses inline shortcodes (CommonLibrary::parseInTextFunctions()), assembles meta tags (Ci4msseoLibrary), and loads categories, tags, authors, breadcrumbs, and comments.
  • SEO fields: Blog and page records store JSON metadata (coverImage, description, keywords).

Media, File Management & Logs

  • Modules\Media integrates elFinder with MIME allowlists, trash handling, and optional WebP conversion via claviska/simpleimage.
  • Modules\Fileeditor offers in-project file browsing/editing secured by realpath checks.
  • Modules\Logs wraps seunmatt/codeigniter-log-viewer so admins can inspect writable/logs/ from /backend/logs without shell access.

CLI & Automation

  • app/Commands/* contains admin generators (make:acontroller, make:amodel, make:abview) and route regeneration (create:route).
  • Module scaffolding is provided by the Composer package ci4-cms-erp/ext_module_generator via php spark make:module.
  • Modules\Methods::moduleScan() inspects the router and aligns permissions with available routes.

Development Tips

  • Extend the backend base controller for consistent session/config/view setup.
  • Update Modules\Methods whenever new secured routes are introduced.
  • Clear caches (php spark cache:clear) after modifying settings, menus, or permissions.
  • Keep theme directories synced with required assets (info.xml, screenshot.png).
  • Captcha auto-disables in development due to environment checks in the login controller.

Common Data Tables

Frequent tables include users, auth_groups, auth_permissions_pages, auth_users_permissions, modules, pages, blog, blog_categories_pivot, tags, tags_pivot, menu, settings, login_rules, and others introduced by modules.

Consult module migrations for schema details.
